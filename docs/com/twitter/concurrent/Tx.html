Tx

Related Docs: object Tx | package concurrent

trait Tx[+T] extends AnyRef

A Tx is used to mediate multi-party transactions with the following protocol:

  1. A transaction is complete when all parties have completed acknowledgment. 2. If any party aborts (nack), the entire transaction is considered aborted. 3. Once a transaction has been acknowledged by a party, that acknowledgment must be honored: The party cannot subsequently nack.

  1. abstract def ack(): Future[Result[T]]

    Acknowledge the transaction, committing the party should the other parties also acknowledge.

    Acknowledge the transaction, committing the party should the other parties also acknowledge. The (potentially delayed) result of the complete transaction is returned upon acknowledgment. A party may not nack() after ack().

  2. abstract def nack(): Unit

    Abort the transaction.

    Abort the transaction. It is invalid to abort a transaction after acknowledging it.
